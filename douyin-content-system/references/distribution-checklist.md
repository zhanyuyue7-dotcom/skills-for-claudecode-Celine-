# 分发检查清单

每条内容发布前必须通过以下检查。这是从"好内容"到"被看见的好内容"的关键步骤。

## 核心指标理解

| 指标 | 算法权重 | 用户的现状 | 需要优化 |
|------|----------|-----------|----------|
| 完播率 | 最高 | 未知，推测中等 | 优化开头30-60秒（长视频标准） |
| 评论率 | 高 | 极低（评论区无人） | **最关键瓶颈** |
| 分享率 | 高 | 低 | 增加社交货币 |
| 点赞率 | 中 | 正常 | 维持 |
| 收藏率 | 中 | 高（与点赞1:1） | 维持 |
| 关注转化 | 中 | 未知 | 通过人设强化 |

## 发布前必查 7 项

### 1. 开头检查（前30-60秒）

- [ ] 前30秒内是否建立了"这个人说的话值得听"的感觉？
- [ ] 是否用了hooks.md中的某种开头方式（可改编）？
- [ ] 是否避免了"大家好，今天来讲一下..."式开头？
- [ ] 开头是否包含**信息差**或**情感共鸣点**？
- [ ] 节奏是否从容自然，而非急促推销？

### 2. 评论触发器检查（最关键）

每条内容至少包含以下之一：

- [ ] **争议点**：说了一个有人会不同意的观点（"我觉得Cursor不如Claude Code"）
- [ ] **二选一提问**：让观众在评论区选边站（"你们觉得XX好还是YY好？"）
- [ ] **经历征集**：让观众分享自己的经历（"你们第一次用AI做出东西是什么时候？"）
- [ ] **故意留白**：没有讲完，让懂的人来补充（"这里还有一个坑，评论区告诉你"）
- [ ] **自嘲/示弱**：暴露弱点引发共鸣或鼓励（"我一个大一的搞这个，是不是在做梦？"）

### 3. 社交货币检查

观众转发你的内容时，他希望自己看起来：

- [ ] **聪明**：转发后朋友觉得"你懂的真多"（适合干货型内容）
- [ ] **前沿**：转发后朋友觉得"你总是知道最新的东西"（适合热点型内容）
- [ ] **有品味**：转发后朋友觉得"你关注的博主很有深度"（适合观点型内容）

至少满足一种社交货币动机。

### 4. 标题/封面检查（图文尤其重要）

- [ ] 标题是否包含具体数字或具体工具名？（"3个" "Claude Code" "5分钟"）
- [ ] 标题是否有信息差或反直觉元素？
- [ ] 封面文字是否在手机上清晰可读（大字、高对比）？
- [ ] 封面是否有视觉焦点（不是一堆文字堆砌）？

### 5. 结尾检查

- [ ] 是否用了4种结尾模板之一？（参见 hooks.md）
- [ ] 结尾是否从"有用"升华到"有意义"？
- [ ] 最后一句话是否有记忆点（能让人复述给朋友听）？
- [ ] 是否自然引导了下一条内容（系列感）？

### 6. 内容完整度检查

- [ ] 口播稿：时长控制在60-120秒？（抖音完播率最优区间）
- [ ] 图文：3-6张图，每张图一个核心信息点？
- [ ] 是否有一个清晰的"观众带走的一件事"（one takeaway）？
- [ ] 文案是否符合人设语气？（参见 persona.md 禁忌清单）

### 7. 发布时机检查

- [ ] 是否在活跃时段发布？（工作日：12:00-13:00 或 20:00-22:00；周末：10:00-12:00 或 20:00-23:00）
- [ ] 如果蹭热点，是否在事件发生后24小时内发布？
- [ ] 发布后30分钟内是否能回复评论？（算法权重：博主回复触发二次推荐）

## 评论区运营技巧

发布后的动作和内容本身一样重要：

1. **自己先评论**：发布后立刻用自己的号留一条有料的评论，降低其他人的评论门槛。自评论模板：
   - 补充型："刚才忘说了，其实还有一个点是..."（引导别人也来补充）
   - 提问型："我自己也还在想一个问题：[相关但没在视频里讲的问题]？"
   - 自嘲型："录完才发现说了个口误，[具体口误]，评论区纠正的算你赢"
   - 预告型："下一期准备讲[XX]，你们想先听哪个方向？"
2. **秒回前10条评论**：算法会监测发布后30分钟内的互动速度
3. **用问题回复评论**：别只说"谢谢"，用反问推动对话（"你也试过吗？效果怎么样？"）
4. **置顶争议评论**：如果有人提出不同观点，置顶它——争议 = 流量
